# Dify Workflow 配置指南

本文档说明如何配置 Dify Workflow 以支持飞书机器人的智能功能。

## 输入变量

在 Dify 中创建 Workflow 类型应用，设置以下输入变量：

- `input_text` (string): 待处理的正文内容
- `source_type` (string): 来源类型 (text/url)

---

## LLM 节点配置

### 节点 1: 内容分类（可选）

**目的**：判断用户意图是查询待办、添加待办还是其他

```
你是一个意图识别助手。请分析用户消息的意图。

用户消息：{{input_text}}

请返回以下意图之一：
- query_todo: 查询待办事项（如"我有什么待办"、"查看待办"）
- add_todo: 添加待办事项（如"明天九点开会"、"定个闹钟"）
- summarize: 需要摘要的信息（如发送了一长段文本或链接）

只返回意图关键词，不要其他内容。
```

### 节点 2: 摘要生成

```
你是一个专业的内容摘要助手。请为以下内容生成100字以内的核心摘要。

要求：
- 提炼核心观点
- 语言精练
- 不使用套话开头
- 如果是简短的任务描述（如"明天九点开会"），摘要可以是任务本身

内容：{{input_text}}
```

### 节点 3: 行动项提取（重要）

```
你是一个行动项提取专家。请从以下内容中提取可执行的行动项。

要求：
1. 每个行动项必须以动词开头
2. 具体可执行可验证
3. 避免模糊表述
4. 为每个行动项评估优先级：high/medium/low
5. 提取1-10个行动项

特别注意：
- 如果用户说的是"明天九点闹钟"、"下周五之前完成报告"等，这是明确的待办事项
- 识别时间表达：明天、下周、X月X日、X点等
- 将时间信息包含在行动项内容中

内容：{{input_text}}
```

### 节点 4: 时间提取（新增）

```
你是一个时间提取专家。请从以下内容中提取截止日期时间。

要求：
1. 识别相对时间：明天、后天、下周、下个月等
2. 识别绝对时间：X月X日、X点、X年X月X日等
3. 转换为标准格式：YYYY-MM-DD
4. 如果没有明确时间，返回 null
5. 时间应该是未来的日期

当前日期：{{current_date}}

内容：{{input_text}}

请返回 JSON 格式：
{
  "due_date": "2026-05-10" 或 null,
  "time_description": "明天九点" 或 null
}
```

### 节点 5: 标签生成

```
你是一个标签分类专家。请为以下内容生成3-5个标签。

要求：
- 覆盖技术领域/主题/场景
- 2-4字或英文单词
- 不用宽泛标签
- 不带#号

内容：{{input_text}}
```

---

## 输出格式（JSON 模式）

```json
{
  "summary": "100字以内核心摘要",
  "action_items": [
    {
      "content": "明天九点闹钟",
      "priority": "medium",
      "due_date": "2026-05-10 09:00"
    }
  ],
  "tags": ["闹钟", "提醒"],
  "intent": "add_todo"
}
```

**due_date 格式说明：**
- 有具体时间：`2026-05-10 09:00`（系统会设置精确提醒）
- 仅有日期：`2026-05-10`（系统仅按日提醒）
- 无时间信息：`null`

---

## 时间解析规则

### 相对时间处理

系统需要能够解析以下相对时间表达：

| 用户表达 | 含义 | 示例转换（假设今天是 2026-05-09）|
|---------|------|--------------------------------|
| 明天 | 次日 | 2026-05-10 |
| 后天 | 两天后 | 2026-05-11 |
| 下周一 | 下个星期一 | 2026-05-12 |
| 下周五 | 下个星期五 | 2026-05-16 |
| 本月底 | 当月最后一天 | 2026-05-31 |
| 下个月初 | 下月第一天 | 2026-06-01 |

### 绝对时间处理

| 用户表达 | 标准格式 |
|---------|---------|
| 5月10日 | 2026-05-10 |
| 5月10号 | 2026-05-10 |
| 2026年5月10日 | 2026-05-10 |

### 时间点处理

如果用户提到具体时间点（如"九点"），可以：
1. 在待办内容中保留时间信息
2. 或在 `due_date` 字段中只包含日期，在 `content` 中包含完整描述

## 测试用例

### 用例 1: 添加简单待办

**输入**：
```
帮我定一个明天九点的闹钟
```

**期望输出**：
```json
{
  "summary": "设置明天九点的闹钟",
  "action_items": [
    {
      "content": "明天九点闹钟",
      "priority": "medium",
      "due_date": "2026-05-10"
    }
  ],
  "tags": ["闹钟", "提醒"],
  "intent": "add_todo"
}
```

### 用例 2: 添加工作待办

**输入**：
```
下周五之前完成项目报告
```

**期望输出**：
```json
{
  "summary": "完成项目报告",
  "action_items": [
    {
      "content": "完成项目报告",
      "priority": "high",
      "due_date": "2026-05-16"
    }
  ],
  "tags": ["工作", "报告"],
  "intent": "add_todo"
}
```

### 用例 3: 信息摘要

**输入**：
```
今天下午3点开会讨论新产品的设计方案，产品经理和设计师都会参加。我们需要确定产品的核心功能和用户界面设计方向。
```

**期望输出**：
```json
{
  "summary": "下午3点召开产品设计会议，确定核心功能和UI方向。",
  "action_items": [
    {
      "content": "参加会议讨论产品设计方案",
      "priority": "high",
      "due_date": "2026-05-09"

      "content": "确定产品核心功能",

      "content": "确定UI设计方向",
      "priority": "medium",

  ],
  "tags": ["会议", "产品设计"],
  "intent": "summarize"

```

## 配置步骤

1. 在 Dify 中创建 Workflow 应用
2. 添加输入变量：`input_text`, `source_type`
3. 按顺序添加上述 LLM 节点
4. 配置输出节点，返回 JSON 格式结果
5. 测试并调优提示词
6. 获取 Workflow ID 并配置到 `.env` 文件

---

## 注意事项

1. **时间解析准确性**：确保 AI 能正确识别相对时间，建议在提示词中传入当前日期
2. **优先级判断**：让 AI 根据任务紧急程度自动判断优先级
3. **意图识别**：准确区分"添加待办"和"查询待办"
4. **多任务处理**：一段文本可能包含多个待办事项，需要全部提取
5. **错误处理**：如果 AI 无法解析时间，允许返回 null

## 进阶功能

### 支持重复任务

```
每天早上9点提醒我吃药
```

可以在输出中增加 `repeat` 字段：

```json
{
  "content": "吃药",
  "priority": "medium",
  "due_date": "2026-05-10",
  "repeat": "daily"
}
```

### 支持任务依赖

```
先完成调研报告，再做PPT
```

输出中可以增加 `depends_on` 字段表示任务依赖关系。

### 支持任务分类

```
工作：完成季度报告
生活：买菜
```

可以在输出中增加 `category` 字段进行分类。
