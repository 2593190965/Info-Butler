# 飞书机器人使用示例

本文档展示飞书机器人的完整使用示例。

---

## 场景 1：添加待办事项

### 用户发送：
```
@机器人 帮我定一个明天九点的闹钟
```

### 机器人回复：
```
✅ 信息处理完成

📝 摘要
设置明天九点的闹钟

🏷️ 标签: #闹钟 #提醒

📋 行动项:
1. 🟡 明天九点闹钟
```

**系统自动完成：**
- ✅ 创建待办事项
- ✅ 设置截止日期为明天
- ✅ 分配中等优先级
- ✅ 明天 9:00 前主动提醒

## 高级用法

### 1. 批量添加待办

```
@机器人 本周需要完成：
1. 周一完成数据库设计
2. 周二完成API开发
3. 周三完成前端页面
4. 周四进行测试
5. 周五上线部署
```

机器人会自动提取 5 个待办事项，并设置相应的截止日期。

### 2. 设置优先级

```
@机器人 【紧急】今天必须完成线上BUG修复
```

机器人会识别"紧急"关键词，设置为高优先级（🔴）。

### 3. 带时间的任务

```
@机器人 明天上午9:30参加产品评审会议，会议室305
```

机器人会提取：
- 待办事项：参加产品评审会议
- 时间：明天上午9:30
- 地点：会议室305

### 4. 重复任务（未来功能）

```
@机器人 每天早上9点提醒我查看邮件
```

系统会创建重复任务，每天 9:00 自动提醒。

## 常见问题

**Q: 如何修改待办事项？**
A: 目前需要通过前端页面修改，后续会支持通过机器人修改。

**Q: 如何删除待办事项？**
A: 通过前端页面删除，或标记为"已完成"/"已忽略"。

**Q: 待办事项支持哪些时间格式？**
A: 支持相对时间（明天、下周）和绝对时间（5月10日、2026-05-10）。

**Q: 如何配置提醒时间？**
A: 在 `.env` 文件中设置 `FEISHU_REMINDER_ADVANCE_DAYS`（默认1天）。

**Q: 支持哪些语言？**
A: 目前支持中文，时间解析优化中。

## 最佳实践

1. **简洁明了**：一句话描述清楚任务
2. **包含时间**：明确截止日期便于排序
3. **具体可执行**：以动词开头，避免模糊表述
4. **适当分类**：添加标签便于后续筛选
5. **定期清理**：及时完成或忽略已处理的待办

## 使用技巧

1. **💡 结合前端**：重要事项在机器人创建后，到前端补充详细信息和标签
2. **🔍 快速查询**：在飞书中随时查看待办列表，无需打开浏览器
3. **⏰ 信任提醒**：系统会主动提醒，不用担心遗忘
4. **📊 定期复盘**：每周查看已完成的待办，总结工作成果
5. **🎯 优先级管理**：高优先级事项优先处理，提高效率
